## Stage 07: Output Formatting & Delivery

### Purpose

This stage produces the final deliverables in the required formats. It transforms the verified outline and data into a polished markdown brief and an HTML slide deck. This ensures the research is presented clearly and professionally to the user.

### Inputs

* The revised outline from Stage 05.
* The verified claims and bibliography from Stage 06.
* The original scope document from Stage 01.
* The tension map from Stage 03.

### Actions

1. Draft the full text of the research brief using the verified outline.
2. Ensure the tone is objective, professional, and easy to read.
3. Insert formatted citations for all factual claims.
4. Include a dedicated section summarizing the tensions and contradictions.
5. Format the brief using clean, standard markdown.
6. Extract the key points from the brief to create content for 5-7 slides.
7. Generate an `index.html` file containing the slide deck presentation.
8. Ensure the HTML deck is responsive and visually clean.
9. Save both the markdown brief and the HTML deck to the workspace.
10. Present the final briefing to the user for approval before marking the research complete.

### Outputs

* A comprehensive research brief in markdown format.
* An `index.html` file containing a 5-7 slide presentation.
* A final summary of the research process and findings.
* All deliverables saved in the designated output directory.
* A recorded user approval or requested revision before completion.

### QA Checklist

* [ ] Is the markdown brief fully drafted and free of unfinished marker text?
* [ ] Are all citations correctly formatted in the text?
* [ ] Does the HTML slide deck contain 5-7 slides?
* [ ] Does the slide deck render correctly without broken elements?
* [ ] Are both the markdown and HTML files saved in the correct location?
* [ ] Does the final output directly answer the original thesis question?
* [ ] Are all `[UNVERIFIED]` claims isolated in the Claims Requiring Verification appendix only, with none in the main narrative body?
* [ ] Has the user approved the final briefing or provided revisions to address?
